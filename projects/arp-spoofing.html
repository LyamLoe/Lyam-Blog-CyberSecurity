ARP Spoofing & Man-in-the-Middle Attack

Experimenter: Lyam • Date: 7 November 2025
Educational Cybersecurity Experiment

1. Pendahuluan

ARP Spoofing adalah teknik serangan pada jaringan lokal (LAN) yang mengeksploitasi kelemahan protokol Address Resolution Protocol (ARP). Protokol ini tidak memiliki mekanisme autentikasi, sehingga mudah dipalsukan.

Dalam serangan ini, penyerang memposisikan dirinya sebagai perantara komunikasi antara korban dan gateway, memungkinkan terjadinya Man-in-the-Middle (MITM) attack.

2. Tujuan Praktikum

  • Memahami cara kerja ARP Spoofing
  • Mempelajari konsep MITM pada jaringan lokal
  • Menguji tool Ettercap dan arpspoof
  • Menganalisis lalu lintas jaringan korban

3. Lingkungan Pengujian

RolePlatform
AttackerKali Linux
VictimHost jaringan lokal
GatewayRouter
Interfacewlan0 / eth0

Seluruh pengujian dilakukan di lingkungan isolated lab dan tidak digunakan pada jaringan publik.

4. Metodologi ARP Spoofing

Penyerang mengirimkan ARP reply palsu kepada korban dan gateway. Akibatnya:

  • Korban mengasosiasikan MAC penyerang sebagai gateway
  • Gateway mengasosiasikan MAC penyerang sebagai korban
  • Seluruh trafik melewati attacker

Karena koneksi tetap berjalan normal, korban tidak menyadari serangan.

5. Implementasi Menggunakan Ettercap

5.1 Instalasi
sudo apt update
sudo apt install ettercap-text-only
5.2 Menjalankan GUI
sudo ettercap -G
5.3 Konfigurasi
  • Sniff → Unified sniffing
  • Pilih interface
  • Scan host
  • Set Target 1 (Victim)
  • Set Target 2 (Gateway)
  • MITM → ARP poisoning
  • Start sniffing

6. Implementasi Menggunakan arpspoof

sudo apt install dsniff
    
sudo arpspoof -i wlan0 -t 192.168.1.100 192.168.1.1
sudo arpspoof -i wlan0 -t 192.168.1.1 192.168.1.100

Dua perintah ini memastikan attacker berada di tengah komunikasi korban dan gateway.

7. Hasil Pengujian

  • Lalu lintas berhasil dialihkan
  • Koneksi korban tetap normal
  • Data dapat dianalisis via packet sniffer

8. Analisis Keamanan

ARP Spoofing membuktikan bahwa jaringan lokal tanpa proteksi sangat rentan terhadap MITM. Ini adalah design flaw, bukan bug implementasi.

9. Mitigasi

  • Static ARP
  • Dynamic ARP Inspection
  • DHCP Snooping
  • VPN
  • IDS/IPS

10. Kesimpulan

Praktikum ini menunjukkan bahwa ARP Spoofing adalah serangan yang mudah dilakukan namun berdampak besar.

Pemahaman teknik ini penting agar sistem dapat dideteksi dan dicegah, bukan disalahgunakan.
